Concepto Base

RAG (Retrieval-Augmented Generation)

RAG es una técnica que combina la potencia de la recuperación de información con la generación de texto fluida.

¿Cómo funciona?

  1. 1 El usuario hace una pregunta.
  2. 2 El sistema busca información relevante en una base de datos.
  3. 3 Esa información se pasa como contexto a un modelo de lenguaje.
  4. 4 El modelo genera una respuesta basada en ese contexto.

Características Clave

  • No depende solo del conocimiento del modelo.
  • Usa información externa (documentos, web, PDFs, etc.).
  • Mejora precisión y reduce errores (alucinaciones).
Ejemplo Real:

"¿Cuál es el horario de atención?"

Busca en documentos → Encuentra el chunk de horario → Genera respuesta precisa.

Estructura

Chunks: Fragmentación Inteligente

Los chunks son fragmentos pequeños de texto en los que se divide un documento grande para su procesamiento.

Búsqueda Ágil

Facilitan la localización precisa de información relevante.

Escalabilidad

Permiten trabajar con textos extremadamente largos.

Precisión

Mejoran drásticamente el enfoque del contexto en RAG.

Buenas Prácticas

200–500 palabras Sentido completo Evitar cortes abruptos
Matemática del Lenguaje

Embeddings: Vectores Semánticos

Representaciones numéricas (vectores) de texto que capturan su significado profundo, no solo las palabras.

[0.12, -0.44, 0.98, 0.01, ...]

Los números representan características abstractas: tema, contexto y matices.

¿En qué se basan?

Se generan mediante IA entrenada con libros, web y artículos para entender:

  • Contexto: Diferencia entre "Banco" (asiento) y "Banco" (finanzas).
  • Relaciones: Entiende que Rey - Hombre + Mujer ≈ Reina.
  • Uso real: Capta cómo fluye el lenguaje humano.

Uso en RAG

  1. Chunks → Embedding
  2. Almacenamiento en DB Vectorial
  3. Pregunta → Embedding
  4. Comparación por similitud (Coseno)
  5. Recuperación del contexto
Servidor

MCP Server

Un servicio que expone datos y herramientas directamente a un modelo de lenguaje.

  • Acceso a documentos dinámicos.
  • Búsqueda en bases de datos vivas.
  • Ejecución de funciones (ej: search_docs).
Cliente

MCP Client

El componente orquestador que conecta al usuario con el servidor y el modelo.

  • Recibe la pregunta del usuario.
  • Consulta al MCP Server.
  • Pasa la info recolectada al modelo.
  • Entrega la respuesta final.
